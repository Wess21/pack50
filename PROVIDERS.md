# LLM Providers Management

## Обзор

Pack50 теперь поддерживает гибкое управление множественными LLM провайдерами через удобный веб-интерфейс. Вы можете добавлять, редактировать и переключаться между различными провайдерами без перезапуска бота.

## Новый интерфейс

Откройте **http://localhost:3000/admin.html** для доступа к улучшенной панели управления.

### Возможности:

✅ **Множественные провайдеры** - Добавляйте неограниченное количество провайдеров
✅ **Модальное окно** - Чистый интерфейс для добавления/редактирования провайдеров
✅ **Список провайдеров** - Визуальный список всех настроенных провайдеров
✅ **Быстрое переключение** - Активируйте нужного провайдера одним кликом
✅ **Тестирование подключения** - Проверка работоспособности перед активацией
✅ **Кастомные эндпоинты** - Поддержка vsellm.ru и других OpenAI-совместимых API

## Структура данных

### Таблица `llm_providers`

```sql
CREATE TABLE llm_providers (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL UNIQUE,           -- Название провайдера
  provider_type VARCHAR(20) NOT NULL,          -- 'anthropic' | 'openai'
  api_key_encrypted TEXT NOT NULL,             -- Зашифрованный API ключ
  encryption_iv TEXT NOT NULL,                 -- IV для расшифровки
  api_base_url TEXT,                           -- Кастомный API URL (опционально)
  model_name TEXT,                             -- Кастомная модель (опционально)
  is_active BOOLEAN DEFAULT FALSE,             -- Только один может быть TRUE
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## API Endpoints

### GET /api/providers
Список всех провайдеров (без API ключей)

**Response:**
```json
[
  {
    "id": 1,
    "name": "VSE LLM - Claude",
    "provider_type": "openai",
    "api_base_url": "https://api.vsellm.ru/v1",
    "model_name": "anthropic/claude-haiku-4.5",
    "is_active": true,
    "created_at": "2026-03-01T10:00:00Z",
    "updated_at": "2026-03-01T10:00:00Z"
  }
]
```

### POST /api/providers
Создать новый провайдер

**Request:**
```json
{
  "name": "VSE LLM - Claude",
  "provider_type": "openai",
  "api_key": "sk-your-key",
  "api_base_url": "https://api.vsellm.ru/v1",
  "model_name": "anthropic/claude-haiku-4.5",
  "is_active": true
}
```

### PUT /api/providers/:id
Обновить провайдер

**Request:**
```json
{
  "name": "VSE LLM - Claude Updated",
  "api_key": "new-key",  // Опционально
  "is_active": true
}
```

### DELETE /api/providers/:id
Удалить провайдер

### POST /api/providers/:id/test
Тестировать подключение к провайдеру

### POST /api/providers/:id/activate
Активировать провайдера (деактивирует всех остальных)

## Примеры использования

### Добавление VSE LLM провайдера

1. Откройте http://localhost:3000/admin.html
2. Войдите (admin / changeme)
3. Нажмите "+ Добавить провайдер"
4. Заполните форму:
   - **Название**: VSE LLM - Claude Haiku
   - **Тип провайдера**: OpenAI Compatible
   - **API Key**: ваш ключ от vsellm.ru
   - **API Base URL**: https://api.vsellm.ru/v1
   - **Модель**: anthropic/claude-haiku-4.5
   - ✅ **Сделать активным провайдером**
5. Нажмите "Сохранить"
6. Нажмите "Тест" для проверки подключения

### Переключение между провайдерами

В списке провайдеров просто нажмите кнопку "Активировать" на нужном провайдере. Бот начнет использовать его для следующих запросов (без перезапуска).

### Добавление нескольких конфигураций

Вы можете добавить несколько провайдеров для разных целей:

- **Anthropic Claude Sonnet 4.5** - для сложных задач
- **VSE LLM - Claude Haiku** - для быстрых ответов (дешевле)
- **OpenAI GPT-4o** - для резервного канала

## Миграция со старой системы

Если у вас уже были настроены провайдеры через старый интерфейс (index.html), они автоматически мигрированы в новую таблицу `llm_providers`.

Старые поля в `bot_config` остались для обратной совместимости, но рекомендуется использовать новый интерфейс.

## Provider Factory Logic

Система создания провайдера работает следующим образом:

1. Ищет активного провайдера в таблице `llm_providers`
2. Если найден - использует его настройки
3. Если нет - fallback на старую систему (`bot_config`)
4. Если и там нет - fallback на переменные окружения (.env)

Это обеспечивает плавный переход и обратную совместимость.

## Безопасность

- API ключи шифруются с помощью AES-256-CBC перед сохранением
- Для каждого ключа генерируется уникальный IV
- Веб-интерфейс защищен JWT аутентификацией
- API ключи никогда не возвращаются в GET запросах

## Troubleshooting

### "Provider connection failed" при тесте

1. Проверьте правильность API ключа
2. Убедитесь что API Base URL корректен
3. Проверьте что модель существует у провайдера
4. Проверьте логи: `tail -f logs/combined.log`

### Бот не отвечает после переключения провайдера

Провайдер создается при каждом запросе, поэтому переключение применяется сразу. Если бот не отвечает:

1. Проверьте что провайдер активен (зеленая плашка "АКТИВЕН")
2. Протестируйте провайдера кнопкой "Тест"
3. Проверьте баланс API ключа у провайдера

### "No active provider found"

Активируйте хотя бы одного провайдера:
1. Откройте список провайдеров
2. Нажмите "Активировать" на нужном провайдере
3. Или создайте нового с галочкой "Сделать активным"
