# Руководство по миграции на новую систему провайдеров

## Что изменилось?

### ✅ Было (старая система):
- Один Anthropic ИЛИ один OpenAI провайдер
- Настройки на главной странице вместе с документами
- Ручное редактирование полей для vsellm.ru

### 🎉 Стало (новая система):
- **Неограниченное количество провайдеров**
- **Модальное окно** для управления каждым провайдером
- **Список всех провайдеров** с визуальными карточками
- **Быстрое переключение** между провайдерами
- **Тестирование** перед активацией
- **Чистый интерфейс** - провайдеры вынесены в отдельную секцию

## Инструкция по миграции

### Шаг 1: Остановить бот (если запущен)

```bash
# Если запущен через npm
# Нажмите Ctrl+C

# Если запущен через Docker Compose
docker compose down
```

### Шаг 2: Применить миграцию базы данных

Миграция уже применена автоматически! Ваши существующие провайдеры из старой системы были перенесены в новую таблицу.

Проверить можно так:

```bash
psql postgresql://pack50:dev_password@localhost:15432/pack50 -c "SELECT * FROM llm_providers;"
```

### Шаг 3: Пересобрать и запустить

```bash
npm run build
npm run dev
```

### Шаг 4: Открыть новый интерфейс

Откройте: **http://localhost:3000/admin.html**

Войдите с теми же учетными данными (admin / changeme)

### Шаг 5: Проверить миграцию провайдеров

В разделе "🔌 LLM Провайдеры" вы должны увидеть:

- Ваши старые провайдеры (если были настроены)
- Один из них отмечен как активный

### Шаг 6: Добавить vsellm.ru провайдера (если нужно)

1. Нажмите "+ Добавить провайдер"
2. Заполните:
   - **Название**: VSE LLM - Claude Haiku
   - **Тип**: OpenAI Compatible
   - **API Key**: ваш ключ от vsellm.ru
   - **API Base URL**: https://api.vsellm.ru/v1
   - **Модель**: anthropic/claude-haiku-4.5
   - ✅ Сделать активным
3. Сохранить
4. Нажать "Тест" для проверки

## Обратная совместимость

### Старый интерфейс всё ещё работает

- **index.html** - старый интерфейс (всё на одной странице)
- **admin.html** - новый интерфейс (рекомендуется)

Оба работают параллельно, но рекомендуется использовать **admin.html**.

### API остался прежним

Все старые API endpoints работают:
- `POST /api/admin/config` - работает
- `GET /api/admin/config` - работает

Но для управления провайдерами используйте новые:
- `GET /api/providers` - список
- `POST /api/providers` - создать
- `PUT /api/providers/:id` - обновить
- `DELETE /api/providers/:id` - удалить
- `POST /api/providers/:id/test` - тест
- `POST /api/providers/:id/activate` - активировать

## Откат на старую систему

Если что-то пошло не так, можно временно вернуться к старой системе:

1. Откройте http://localhost:3000/index.html
2. Используйте старые поля для настройки провайдеров

Provider Factory автоматически использует fallback:
1. Сначала ищет в `llm_providers` (новая таблица)
2. Если нет - ищет в `bot_config` (старая система)
3. Если нет - использует .env файл

## Часто задаваемые вопросы

### Нужно ли удалять старых провайдеров из bot_config?

Нет, они остаются для обратной совместимости. Но новые провайдеры из `llm_providers` имеют приоритет.

### Можно ли использовать оба интерфейса одновременно?

Да, но не рекомендуется. Используйте только **admin.html** для предсказуемого поведения.

### Что будет с моими API ключами?

Они автоматически мигрированы в новую таблицу в зашифрованном виде. Безопасность не изменилась.

### Как переключиться между провайдерами?

В новом интерфейсе просто нажмите "Активировать" на нужном провайдере. Изменения применяются мгновенно без перезапуска бота.

### Сколько провайдеров можно добавить?

Неограниченно. Но только один может быть активным в любой момент времени.
