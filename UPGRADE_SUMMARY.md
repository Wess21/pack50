## 📦 Что установлено

### Новая структура базы данных

```sql
CREATE TABLE llm_providers (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL UNIQUE,
  provider_type VARCHAR(20) NOT NULL,
  api_key_encrypted TEXT NOT NULL,
  encryption_iv TEXT NOT NULL,
  api_base_url TEXT,
  model_name TEXT,
  is_active BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Новые API endpoints

```
GET    /api/providers           - Список провайдеров
POST   /api/providers           - Создать провайдер
GET    /api/providers/:id       - Детали провайдера
PUT    /api/providers/:id       - Обновить провайдер
DELETE /api/providers/:id       - Удалить провайдер
POST   /api/providers/:id/test  - Тест подключения
POST   /api/providers/:id/activate - Активировать
```

## 🎯 Как использовать

### Шаг 1: Бот уже готов к работе!

Миграция была применена автоматически. Если у вас были настроены провайдеры, они уже перенесены в новую систему.

### Шаг 2: Откройте новый интерфейс

```bash
# Убедитесь что бот запущен
npm run dev

# Откройте в браузере
open http://localhost:3000/admin.html
```

### Шаг 3: Добавьте vsellm.ru провайдер

1. Нажмите "+ Добавить провайдер"
2. Заполните форму:
   - **Название:** VSE LLM - Claude Haiku
   - **Тип:** OpenAI Compatible
   - **API Key:** ваш ключ
   - **API Base URL:** https://api.vsellm.ru/v1
   - **Модель:** anthropic/claude-haiku-4.5
   - ✅ **Сделать активным**
3. Сохранить
4. Нажать "Тест"

### Шаг 4: Протестируйте в Telegram

Отправьте боту сообщение - он должен ответить используя нового провайдера!

## 📊 Что изменилось технически

### Backend

1. **Новая таблица БД:** `llm_providers`
2. **Новый роут:** `src/api/routes/providers.ts` (350+ строк)
3. **Обновлен:** `src/services/llm/provider-factory.ts` - fallback логика

5. **Обновлен:** `src/index.ts` - регистрация нового роута

### Frontend

1. **Создан:** `public/admin.html` - новый интерфейс (700+ строк)
2. **Сохранен:** `public/index.html` - старый интерфейс (для обратной совместимости)

### База данных

1. **Создана таблица:** `llm_providers` с шифрованием ключей
2. **Добавлены колонки:** `api_base_url`, `llm_model_name` в `bot_config`
3. **Миграция данных:** Существующие провайдеры автоматически перенесены

## 🔄 Обратная совместимость

### Старый интерфейс работает

Если хотите вернуться к старому интерфейсу:
```
http://localhost:3000/index.html
```

### Provider Factory Fallback

Провайдеры загружаются в порядке приоритета:
1. ✅ `llm_providers` таблица (новая система)
2. ✅ `bot_config` таблица (старая система)
3. ✅ `.env` файл (переменные окружения)

Если что-то не работает - система автоматически переключится на резервный вариант.
