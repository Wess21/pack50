# Pack50 Деплой и Обновление

Быстрое руководство по автоматизированному развертыванию Pack50 бота на новом VDS сервере и его обновлению без даунтайма.

## 1. Развертывание на новом VDS

### Требования к серверу
- **OS:** Ubuntu 20.04/22.04 или Debian 11/12
- Чистый сервер с доступом по SSH (желательно root)

### Инструкция по установке

1. Подключитесь к вашему серверу:
   ```bash
   ssh root@your-server-ip
   ```

2. Скачайте скрипт развертывания:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/Wess21/pack50/main/deploy-vds.sh -o deploy-vds.sh
   chmod +x deploy-vds.sh
   ```

3. Запустите автоматический деплой:
   ```bash
   ./deploy-vds.sh
   ```

**Что делает скрипт:**
- Автоматически установит Docker и другие необходимые утилиты.
- Создаст папку `~/pack50-bot` и сгенерирует `.env` файл со случайными безопасными паролями для БД.
- Скачает свежий билд бота из репозитория GitHub Container Registry (ghcr.io) и запустит все контейнеры.

### Важно: После установки
Бот запустится с временным токеном-заглушкой. Вам нужно установить свой рабочий токен от [@BotFather](https://t.me/BotFather):
```bash
cd ~/pack50-bot
nano .env  # Установите ваш реальный BOT_TOKEN
docker compose up -d  # Перезапустите контейнеры
```
Данные администратора для входа в панель будут сохранены в `~/pack50-bot/admin_credentials.txt`.

---

## 2. Обновление бота на VDS (Zero-Downtime)

Вся логика сборки происходит в GitHub Actions. При пуше в ветку `main`, GitHub собирает новый Docker образ. Чтобы применить эти обновления на сервере:

1. Подключитесь к вашему серверу по SSH.

2. Запустите скрипт обновления в папке бота:
   ```bash
   cd ~/pack50-bot
   bash ./deploy-vds.sh
   ```

**Как это работает:**
- Скрипт просто скачает обновленный образ из GitHub Registry без остановки остальных компонентов инфраструктуры.
- Контейнер бота будет пересоздан (мягкий перезапуск с нулевым даунтаймом базы данных).
- Все новые миграции базы данных будут применены ботом автоматически при запуске.
