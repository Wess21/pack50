# AI Assistant Box

## What This Is

Коробочное решение AI-ассистент для малого и среднего бизнеса. Telegram-бот с RAG, который клиент разворачивает на своем сервере за 3 команды. Бот отвечает на вопросы из документов бизнеса, собирает данные о клиентах, создает заявки и интегрируется с CRM через вебхуки.

## Core Value

Умный AI-ассистент, который действительно продуктивно работает на бизнес — качественно отвечает на вопросы клиентов, помнит контекст диалогов, проактивно ведет к решению, снижает нагрузку на операторов и увеличивает конверсию в заявки.

### Out of Scope

- WhatsApp интеграция — v2 (фокус на MVP с Telegram)
- Max (VK) интеграция — v2
- Входящие вебхуки (внешняя система отправляет сообщения боту) — v2
- Продвинутая аналитика и дашборды — базовых метрик достаточно для v1
- Мультитенант архитектура — безопасность требует изоляции данных

## Context

**Целевая аудитория:**
Малый и средний бизнес в РФ (локальные магазины, сервисы, интернет-магазины, B2B компании) — любой бизнес, которому нужен AI-бот для коммуникации с клиентами.

**Мотивация проекта:**
Хайп вокруг AI-технологий и возможность монетизировать спрос на AI-ассистентов. Цель — запустить AI-компанию с продаваемым продуктом.

**Бизнес-модель:**
Лицензия + платная поддержка. Опционально — предоставление доступа к AI-моделям за отдельную плату (если клиент не хочет использовать свой API-ключ).

**Позиционирование:**
"Умные автоответы с AI" — клиент получает работающего бота за неделю после покупки. Быстрое развертывание, автономная работа, безопасность данных.

**Критичность безопасности:**
- 152-ФЗ (персональные данные клиентов)
- Конфиденциальная бизнес-информация (прайсы, процессы, внутренние данные)
- Недоверие российских компаний к зарубежным облачным решениям
- Требование локального хранения данных на сервере клиента

**Технический уровень клиента:**
Клиент не очень технически подкован. Нужна максимально простая инструкция по установке — "скопируй 3 команды в терминал".

## Constraints

- **Tech stack**: Python + FastAPI, LangChain + pgvector, PostgreSQL, Redis, React, Docker Compose — для быстрой разработки и легковесности
- **Производительность**: Должно быстро работать и потреблять мало ресурсов (запуск на небольших VPS у клиентов)
- **Развертывание**: Максимум 3 команды для установки — клиент должен развернуть без глубоких технических знаний
- **Безопасность**: Отдельный инстанс на каждого клиента (не мультитенант) — изоляция данных
- **Timeline**: MVP для демонстрации потенциальным клиентам — прототип, который можно показать и продать
