  <action>
Create LLMService class wrapping Anthropic SDK with streaming support and comprehensive error handling.

**Implementation:**
- Use @anthropic-ai/sdk (already installed) with claude-sonnet-4-5-20250929 model
- Initialize Anthropic client with env.ANTHROPIC_API_KEY and maxRetries: 2
- Implement generateResponse() method accepting { messages, systemPrompt, maxTokens }
- Use streaming API (client.messages.stream()) to avoid timeouts on long responses
- Extract text from message.content[0] (handle text type)
- Log token usage via message.usage.input_tokens and message.usage.output_tokens
- Implement handleError() method with specific error handling:
  - 400: "Извините, ваш запрос не может быть обработан. Попробуйте переформулировать."
  - 401: "Ошибка конфигурации AI. Пожалуйста, свяжитесь с администратором."
  - 429: "Сейчас высокая нагрузка на AI. Пожалуйста, попробуйте через минуту."
  - 500/529: "AI временно недоступен. Пожалуйста, попробуйте позже или обратитесь к оператору."
  - default: "Произошла ошибка при генерации ответа. Пожалуйста, попробуйте снова."
- Use Anthropic.APIError type checking (not string matching on error messages)

**Pattern from research:** Follow Pattern 1 (LLM Service with Streaming) from 03-RESEARCH.md

**Types:**
```typescript
interface LLMRequest {
  messages: Anthropic.MessageParam[];
  systemPrompt: string;
  maxTokens?: number;
}
```

**Why streaming:** Prevents timeout for responses >30 seconds, provides better UX opportunity for typing indicators in Phase 4.

**Why not @anthropic-ai/tokenizer:** Inaccurate for Claude 3+, use API response usage field instead (per research).

Create ContextManager class that dynamically allocates token budget between system prompt, RAG context, and conversation history.

**Implementation:**
- Define ContextBudget interface: { systemPrompt, ragContext, conversationHistory, maxOutput, total }
- Set MODEL_CONTEXT_LIMIT = 200000 (Claude Sonnet 3.5+)
- Set SAFE_USAGE_PERCENT = 0.80 (stay under 80% per requirement LLM-05)
- Implement calculateBudget(ragChunks: number, conversationTurns: number): ContextBudget
  - Fixed allocations: systemPrompt = 800 tokens, maxOutput = 1024 tokens
  - Remaining budget split between RAG and history
  - For conversational queries (>3 turns): 50/50 split
  - For simple queries (≤3 turns): 70/30 split (prioritize RAG context)
- Implement truncateHistory(messages: Anthropic.MessageParam[], maxTokenBudget: number)
  - Rough estimate: 1 token ≈ 4 characters
  - Keep most recent messages (reverse iteration)
  - Preserve user/assistant alternation
  - Return truncated array

**Pattern from research:** Follow Pattern 2 (Context Window Management) from 03-RESEARCH.md

**Why 80% limit:** Prevents edge-case context overflow, accounts for tokenization variance (per research pitfall 1).

**Why dynamic split:** Conversational queries need more history for pronoun resolution, simple queries need more RAG context for accuracy.

**system-prompts.ts:**
- Define SYSTEM_PROMPTS object with three personas: consultant, support, orderTaker
- Each prompt defines: goals (5 numbered items), behavior rules (4 bullet points), current date placeholder
- Consultant persona:
  - Goals: Answer questions using docs, ask clarifying questions, guide to solutions, collect contact info naturally, cite sources
  - Behavior: Professional but friendly, concise (2-3 paragraphs), admit when uncertain, proactively suggest next steps
- Support persona:
  - Goals: Diagnose issues using docs, step-by-step troubleshooting, collect error details, escalate when needed, reference docs
  - Behavior: Patient and empathetic, simple language, confirm understanding, document issues
- OrderTaker persona:
  - Goals: Help find products, collect order details (items/quantities/address), confirm pricing/payment, generate order data, answer product questions
  - Behavior: Efficient and clear, summarize before confirmation, clarify ambiguities, provide delivery estimates
- Export buildSystemPrompt(persona: keyof typeof SYSTEM_PROMPTS): string
  - Replace {date} placeholder with current date (YYYY-MM-DD format)

**prompt-builder.ts:**
- Export buildPrompt(ragContext: string, conversationHistory: Anthropic.MessageParam[], userQuery: string)
- Assemble messages array: [...conversationHistory, { role: 'user', content: `Context from knowledge base:\n\n${ragContext}\n\n---\n\nUser question: ${userQuery}` }]
- Place RAG context at end before user query (avoid lost-in-middle problem per research)

**Pattern from research:** Follow Pattern 3 (System Prompt Templates) from 03-RESEARCH.md

**Why three personas:** Phase 4 will make this configurable, establishing pattern now. Consultant is default for Phase 3.

**Why Russian prompts:** Could be Russian or English - LLM will respond in appropriate language based on user input. English prompts are clearer for instruction-following.
