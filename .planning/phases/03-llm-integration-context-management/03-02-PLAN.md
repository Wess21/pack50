  <action>
Create WebhookService class with axios-retry exponential backoff for reliable CRM delivery.

**webhook.ts implementation:**
- Define WebhookPayload interface:
  - event_type: 'lead_collected' | 'conversation_complete' | 'escalation'
  - timestamp: string (ISO 8601)
  - webhook_id: string (UUID for idempotency)
  - user_id: string (Telegram user ID)
  - username?: string (Telegram username if available)
  - message: string (latest user message or summary)
  - conversation_summary?: string (optional)
  - collected_data: { name?, email?, phone?, company?, [key: string]: any }
  - tags?: string[] (categorization)
  - lead_score?: number (optional quality score)
- Create WebhookService class constructor accepting webhookUrl parameter
- Initialize axios client with 10s timeout, Content-Type: application/json, User-Agent: AI-Assistant-Box/1.0
- Configure axios-retry:
  - retries: 5
  - retryDelay: axiosRetry.exponentialDelay
  - retryCondition: network errors OR status 408, 429, 500+
  - shouldResetTimeout: true
- Implement send(payload: WebhookPayload): Promise<void>
  - Log attempt: url, eventType, userId
  - POST payload to webhook URL
  - Log success: status, userId
  - Catch errors, log failure details
  - Throw error if all retries exhausted
- Handle undefined WEBHOOK_URL gracefully (log warning, skip delivery)

**env.ts changes:**
- Add WEBHOOK_URL to env schema: z.string().url().optional()
- Add validation for URL format if provided

**.env.example changes:**
- Add: `# Webhook Configuration\nWEBHOOK_URL=https://your-crm-system.com/webhook/endpoint`
- Add comment: `# Optional - leave empty to disable webhook delivery`

**Pattern from research:** Follow Pattern 4 (Webhook Delivery with Retry) from 03-RESEARCH.md

**Why 5 retries:** Exponential backoff with 5 retries covers ~30 seconds of downtime (enough for transient issues, not hours of outage).

**Why 10s timeout:** Prevents blocking user response while still allowing reasonable CRM processing time (per research pitfall 6).

**Why UUID webhook_id:** Enables CRM systems to implement idempotent processing (ignore duplicates if retry delivers twice).
