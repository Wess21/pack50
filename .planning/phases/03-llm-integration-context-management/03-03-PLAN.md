  <name>Task 1: Update session type and message handler with LLM integration</name>
  <files>
    src/types/session.ts
    src/bot/handlers/message-handler.ts
  </files>
  <action>
Extend session storage to track conversation history and integrate LLM response generation into message handler.

**session.ts changes:**
- Add conversationHistory field to SessionData interface: `conversationHistory: Array<{ role: 'user' | 'assistant', content: string, timestamp: string }>`
- Initialize conversationHistory as empty array in default session

**message-handler.ts changes:**
- Import: LLMService, ContextManager, WebhookService, buildSystemPrompt, buildPrompt, searchDocuments, extractContext
- Initialize services at module level:
  - const llmService = new LLMService()
  - const contextManager = new ContextManager()
  - const webhookService = new WebhookService(env.WEBHOOK_URL || '')
- Modify message handling flow:
  1. Retrieve RAG context: `const ragResults = await searchDocuments(messageText, { k: 5, minSimilarity: 0.3 })`
  2. Extract context: `const ragContext = extractContext(ragResults)`
  3. Load conversation history from session.conversationHistory (convert to Anthropic.MessageParam format)
  4. Calculate token budget: `const budget = contextManager.calculateBudget(ragResults.length, session.conversationHistory.length)`
  5. Truncate history: `const truncatedHistory = contextManager.truncateHistory(historyAsMessages, budget.conversationHistory)`
  6. Truncate RAG context: `ragContext.substring(0, budget.ragContext * 4)`
  7. Build prompt: `const messages = buildPrompt(truncatedRagContext, truncatedHistory, messageText)`
  8. Generate response: `const response = await llmService.generateResponse({ messages, systemPrompt: buildSystemPrompt('consultant'), maxTokens: budget.maxOutput })`
  9. Update session history: push user message and assistant response
  10. Send response to user via ctx.reply()
  11. If collected_data detected in session (e.g., name + email + phone all present), send webhook
- Wrap LLM call in try/catch:
  - On error: log error, send user-friendly fallback message, return gracefully
  - Don't crash bot on LLM failures
- Keep existing RAG citation formatting for debugging (optional: include in response footer)

**Session history management:**
- Store last 10 turns maximum in session (older messages dropped to save Redis memory)
- Include timestamp in each history entry for debugging
- Convert session history format to Anthropic.MessageParam for LLM calls

**Webhook trigger logic:**
- Check if session.collected_data has name, email, and phone (basic lead qualification)
- If complete: build WebhookPayload with event_type: 'lead_collected', send via webhookService.send()
- Log webhook delivery (success or failure), but don't block user response
- Webhook errors logged as warnings, not errors

**Pattern from research:** Follow multi-turn conversation pattern from 03-RESEARCH.md code examples

**Why consultant persona:** Default for Phase 3. Phase 4 will make configurable.

**Why 10 turn limit:** Balances conversation continuity with Redis memory usage (per research, most conversations <10 turns).
