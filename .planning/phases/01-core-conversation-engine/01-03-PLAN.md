    - name?: string
    - email?: string
    - phone?: string

    Create EMAIL_REGEX: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
    Create PHONE_REGEX: /^[\+]?[(]?[0-9]{1,3}[)]?[-\s\.]?[(]?[0-9]{1,4}[)]?[-\s\.]?[0-9]{1,4}[-\s\.]?[0-9]{1,9}$/

    Implement extractDataFromMessage(message: string, existingData: Partial ExtractedData): Promise Partial ExtractedData:

    1. Extract email if missing:
       - Test message against EMAIL_REGEX
       - If match found and valid, add to extracted object

    2. Extract phone if missing:
       - Test message against PHONE_REGEX
       - If match found, add to extracted object

    3. Extract name via LLM if missing and no email/phone detected:
       - Import Anthropic SDK, create client with env.ANTHROPIC_API_KEY
       - Build prompt: "Extract the person's name from this message. Return only the name, or 'NONE' if no name present.\n\nMessage: \"${message}\""
       - Call anthropic.messages.create with model "claude-sonnet-4.5-20250514", max_tokens 50
       - Parse response text, if not "NONE" add to extracted.name
       - Handle API errors gracefully (return empty extraction, log error)

    4. Return extracted data object

    Follow research Code Examples > Data Extraction with Regex + LLM Fallback. Use Claude for complex name patterns (Jr., titles, Cyrillic names) that regex can't handle reliably.

    - Implement manageConversationContext(session: SessionData): { messages: Message[], summary?: string }
      - MAX_CONTEXT_MESSAGES = 10
      - If messageHistory.length <= 10: return { messages: history }
      - Else: take last 10 messages as recentMessages, older as oldMessages
      - If conversationSummary not exists or oldMessages > 20: generate summary via Claude API
        - Prompt: "Summarize the key points and context from this conversation concisely (2-3 sentences):\n\n{formatted messages}"
        - Store in session.conversationSummary
      - Return { messages: recentMessages, summary: conversationSummary }

    - Implement isLeadComplete(leadData: Partial LeadData): boolean
      - Return Boolean(leadData.name && leadData.email && leadData.phone)

    - Export functions

    **src/bot/keyboards.ts:**
    - Import InlineKeyboard from grammy
    - Create confirmKeyboard: InlineKeyboard with buttons "✓ Confirm" (callback_data: "confirm_lead"), "✗ Edit" (callback_data: "edit_lead")
    - Export confirmKeyboard

    **src/bot/conversations/lead-collection.ts:**
    - Import Conversation, ConversationFlavor from @grammyjs/conversations
    - Import MyContext, extractDataFromMessage, manageConversationContext, isLeadComplete, confirmKeyboard
    - Import repositories: findOrCreateUser, createConversation, addMessage, updateConversationLeadData

    Define async leadCollectionFlow(conversation: Conversation MyContext, ctx: MyContext): Promise void

    Flow logic:
    1. Welcome: await ctx.reply("Привет! Я помогу вам оставить заявку. Как вас зовут?")

    2. Loop until lead complete (while !isLeadComplete(ctx.session.leadData)):
       a. Wait for user message: const msgCtx = await conversation.waitFor("message:text")
       b. Extract data: await conversation.external(() => extractDataFromMessage(msgCtx.message.text, ctx.session.leadData))
       c. Update session: await conversation.external(() => { ctx.session.leadData = {...ctx.session.leadData, ...extractedData} })
       d. Add to message history: ctx.session.messageHistory.push({ role: "user", content: msgCtx.message.text, timestamp: new Date() })
       e. Manage context: { messages, summary } = await conversation.external(() => manageConversationContext(ctx.session))
       f. Generate next question based on missing fields:
          - If no name: "Отлично! А как мне к вам обращаться?"
          - If no email: "Прекрасно! Какой у вас email?"
          - If no phone: "Последний вопрос - ваш номер телефона?"
       g. Reply with next question
       h. Add assistant message to history

    3. Confirmation: Build confirmation message with collected data, show confirmKeyboard
       - await ctx.reply(`Пожалуйста, подтвердите информацию:\n\nИмя: ${name}\nEmail: ${email}\nТелефон: ${phone}`, { reply_markup: confirmKeyboard })

    4. Wait for callback: const callbackCtx = await conversation.waitFor("callback_query:data")

    5. Answer callback query: await callbackCtx.answerCallbackQuery()

    6. Handle confirmation:
       - If "confirm_lead":
         - await conversation.external(async () => {
             const user = await findOrCreateUser(ctx.from.id, ctx.from.username, ctx.from.first_name, ctx.from.last_name);
             const conv = await createConversation(user.id);
             for (const msg of ctx.session.messageHistory) {
               await addMessage(conv.id, msg.role, msg.content);
             }
             await updateConversationLeadData(conv.id, ctx.session.leadData);
           })
         - await ctx.reply("Спасибо! Ваша заявка принята.")
         - Reset session: ctx.session.conversationState = "idle", ctx.session.leadData = {}
       - Else: await ctx.reply("Хорошо, начнем заново. Используйте /start")

    Export leadCollectionFlow

    **src/bot/index.ts:**
    - Import createConversation, conversations from @grammyjs/conversations
    - Import leadCollectionFlow
    - Add conversations plugin: bot.use(conversations())
    - Register conversation: bot.use(createConversation(leadCollectionFlow))
    - Update /start handler to enter conversation: await ctx.conversation.enter("leadCollectionFlow")

    CRITICAL: Wrap ALL side effects (DB writes, API calls, session updates) in conversation.external() to avoid replay bugs (see research Pitfall 1).

    Follow research Pattern 1: Conversation Flow with Slot Filling and Pattern 4: Claude API Integration.
