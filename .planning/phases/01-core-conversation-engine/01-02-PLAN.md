<objective>
Implement stateful conversation management with Redis-backed sessions and core command handlers.

Purpose: Enable bot to remember conversation context across messages and restarts. This is the foundation for multi-turn dialogues - without session persistence, every message is isolated and users must repeat themselves.

Output: Bot that maintains user-specific conversation state in Redis, handles essential commands, and logs activity for debugging.
</objective>
